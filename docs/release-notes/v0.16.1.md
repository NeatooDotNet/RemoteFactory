# v0.16.1 - .NET 10 Package Version Fix

**Release Date:** 2026-03-02
**NuGet:** [Neatoo.RemoteFactory 0.16.1](https://nuget.org/packages/Neatoo.RemoteFactory/0.16.1)

## Overview

Fixes net10.0 targets being forced to use v9 Microsoft.Extensions packages instead of their native v10 versions.

## What's New

None

## Breaking Changes

None

## Bug Fixes

- **net10.0 now uses native v10 packages**: Previously, net10.0 shared a condition with net9.0 (`net9.0 Or net10.0`), forcing it to resolve v9.0.x versions of `Microsoft.Extensions.DependencyInjection`, `Microsoft.Extensions.Logging`, `Microsoft.AspNetCore.Components.WebAssembly`, and related packages. Each target framework now has its own `ItemGroup` with the correct version.
- **Central package versions updated to v10**: `Directory.Packages.props` now uses v10 as the default, with `VersionOverride` for net8.0 and net9.0. This ensures the latest framework always uses the central version without overrides.
- **Removed unused `.v8`/`.v10` alias entries** from `Directory.Packages.props` that were never referenced from any project file.
