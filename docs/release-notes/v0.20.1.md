# v0.20.1 - Internal Methods on Public Factory Interfaces

**Release Date:** 2026-03-07
**NuGet:** [Neatoo.RemoteFactory 0.20.1](https://nuget.org/packages/Neatoo.RemoteFactory/0.20.1)

## Overview

Fixes a bug where internal factory methods were excluded entirely from public factory interfaces in mixed-visibility scenarios, causing CS1061 errors when other domain code called those methods through the interface.

## What's New

None

## Breaking Changes

None

## Bug Fixes

- **Internal methods now appear on public factory interfaces with the `internal` modifier.** Previously, when a factory had mixed visibility (e.g., public `[Create]` + internal `[Fetch]`/`[Insert]`/`[Update]`/`[Delete]`), internal methods were silently dropped from the generated interface. This caused CS1061 compile errors when parent entities called `factory.Save(child)` or `factory.Fetch(id)` through the factory interface. Internal methods are now emitted with the `internal` access modifier on the interface, making them accessible to same-assembly callers while remaining hidden from external consumers.

**Before (broken):**
```csharp
public interface IPersonPhoneFactory
{
    PersonPhone Create();
    // Fetch and Save were missing — CS1061 when called through interface
}
```

**After (fixed):**
```csharp
public interface IPersonPhoneFactory
{
    PersonPhone Create();
    internal PersonPhone Fetch(int id);
    internal PersonPhone? Save(PersonPhone target);
}
```

All-internal interfaces (where every method is internal) are unchanged — the interface itself is `internal`, so no per-method modifier is needed.
