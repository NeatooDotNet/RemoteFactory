# v1.6.0 — NF0106 Diagnostic + Interface-Factory `[AuthorizeFactory<T>]` Pedagogy

**Release Date:** 2026-04-25
**NuGet:** [Neatoo.RemoteFactory 1.6.0](https://nuget.org/packages/Neatoo.RemoteFactory/1.6.0)
**Breaking changes:** None.

---

## Overview

Adds **NF0106**, a new generator diagnostic that catches a previously cryptic misuse: putting a factory-operation attribute (`[Create]`, `[Fetch]`, `[Insert]`, `[Update]`, `[Delete]`, `[Execute]`) on an interface factory method. Previously this surfaced as duplicate-codegen errors (CS0111 / CS0738); now it surfaces as one clear diagnostic at the source location.

Also fills a gap in the Design source of truth: interface-factory `[AuthorizeFactory<T>]` was proven by tests but not demonstrated in the Design project, leading anyone reasoning from the Design code to invent incorrect class-factory CRUD models. The new `AuthorizedRepository.cs` and skill section show the shipped model — `Execute` / `Read` scope, parameter matching, plain impl — backed by compiled snippets.

---

## What's New

### NF0106 — `[Create]`/`[Fetch]`/etc. on interface factory method (Error)

Fires when a factory-operation attribute is placed on a method declared on a `[Factory]` interface. Replaces cryptic CS0111 / CS0738 duplicate-codegen errors with a single targeted diagnostic.

```csharp
// Before — produced CS0111/CS0738 from generated duplicate methods
[Factory]
public partial interface IPersonRepository
{
    [Fetch]   // NF0106 in v1.6.0
    Task<Person> Fetch(int id);
}
```

Interface factories should declare plain method signatures; the operation attributes (`[Create]`, `[Fetch]`, etc.) belong on the implementing class. The diagnostic fires whether or not `[AuthorizeFactory<T>]` is also present, and is keyed off `methodSymbol.ContainingType`, so paired interface/class factories don't misfire.

### Interface-factory `[AuthorizeFactory<T>]` pedagogy

The Design project now ships `AuthorizedRepository.cs` and `InterfaceFactoryAuthorizationTests.cs` (10 tests) demonstrating the supported authorization model on interface factories: `AuthorizeFactoryOperation.Execute` / `Read` scoping, target-parameter matching, and plain implementing class. `CLAUDE-DESIGN.md` gains a new Pattern 2a section, decision-table rows, and updated Anti-Pattern 2 / Critical Rule 4 references.

The RemoteFactory skill mirrors this with a new interface-factory auth section sourced from compiled samples in the reference app via MarkdownSnippets.

## Migration Guide

No action required. If you have an existing project that places `[Create]` / `[Fetch]` / `[Insert]` / `[Update]` / `[Delete]` / `[Execute]` on an interface method, you'll now see **NF0106** instead of CS0111 / CS0738. The fix is the same: move the operation attribute to the implementing class method.
