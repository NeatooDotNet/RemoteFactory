# v10.8.0 - Optional CancellationToken on Factory Methods

**Release Date:** 2026-01-13
**NuGet:** [Neatoo.RemoteFactory 10.8.0](https://nuget.org/packages/Neatoo.RemoteFactory/10.8.0)

## Overview

All generated factory methods now include an optional `CancellationToken` parameter with a default value, making cancellation support available without requiring changes to existing code.

## What's New

### Optional CancellationToken on All Factory Methods

Factory methods now always include `CancellationToken cancellationToken = default` as the last parameter:

```csharp
// Before: CancellationToken only available if domain method had it
var person = await personFactory.Fetch(id);

// After: CancellationToken always available with optional default
var person = await personFactory.Fetch(id); // Still works
var person = await personFactory.Fetch(id, cancellationToken); // Now available
```

This applies to all factory method types:
- `Fetch`, `Create` methods
- `Save`, `TrySave` methods
- `Can*` authorization check methods
- Interface factory methods

### Improved Test Coverage

Added `AuthorizationEnforcementTests` that verify authorization is properly enforced for both class-based and interface-based factories side-by-side.

## Breaking Changes

None. The CancellationToken parameter is optional with a default value, so existing code continues to work unchanged.

## Bug Fixes

None.

## Migration Guide

No migration required. Existing code continues to work. To take advantage of cancellation support:

```csharp
// Pass cancellation token when you have one
var cts = new CancellationTokenSource(TimeSpan.FromSeconds(30));
var person = await personFactory.Fetch(id, cts.Token);
```
