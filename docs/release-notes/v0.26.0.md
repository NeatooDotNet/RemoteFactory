# v0.26.0 - LazyLoad&lt;T&gt; Inheritable for Subclassing

**Release Date:** 2026-03-29
**NuGet:** [Neatoo.RemoteFactory 0.26.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.26.0)

## Overview

Makes `LazyLoad<T>` inheritable so downstream libraries (e.g., Neatoo) can subclass it instead of duplicating the core loading logic.

## What's New

- **`ILazyLoadDeserializable` is now public** -- Enables subclasses to participate in the deserialization merge protocol. The interface remains in the `Neatoo.RemoteFactory.Internal` namespace to signal "use with care."
- **`protected Task<T?>? LoadTask`** -- Read-only access to the in-flight load task, enabling subclass implementations like `WaitForTasks()`.
- **`protected void ClearLoadError()`** -- Allows subclasses to clear the load error state (e.g., for `ClearAllMessages()` / `ClearSelfMessages()`).
- **`protected virtual void OnPropertyChanged(string)`** -- Standard INPC pattern. Subclasses can fire `PropertyChanged` for additional meta-properties and override notification behavior.

## Breaking Changes

None. All changes are strictly additive (widening visibility). `ILazyLoadDeserializable` was previously `internal`, so no external consumer could reference it.
