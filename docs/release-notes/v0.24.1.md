# v0.24.1 - Fix Missing global:: in Generated Type References

**Release Date:** 2026-03-26
**NuGet:** [Neatoo.RemoteFactory 0.24.1](https://nuget.org/packages/Neatoo.RemoteFactory/0.24.1)

## Overview

The source generator now emits `global::` prefixed fully-qualified type references in all generated code, following standard Roslyn source generator best practices. This prevents compilation errors when a class name matches a namespace segment.

## Bug Fixes

- **Fixed namespace resolution ambiguity in generated code.** The generator emitted type references like `Person.Ef.PersonEntity` without the `global::` prefix. When a class name in scope matched a namespace segment (e.g., class `PersonModel` in namespace `Person.DomainModel`), C# resolved the leading `Person` as the class rather than the namespace, causing compilation errors. All generated fully-qualified type references now use `global::` prefix.

- **Fixed inner nullable annotation preservation.** When switching to `SymbolDisplayFormat.FullyQualifiedFormat` for property type extraction, inner nullable annotations on generic types (e.g., `List<string?>`) were being stripped. A custom format (`FullyQualifiedFormatWithNullable`) preserves these annotations.
