# v0.20.0 - Serialization Property Rename & Documentation

**Release Date:** 2026-03-07
**NuGet:** [Neatoo.RemoteFactory 0.20.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.20.0)

## Overview

Renames misleading serialization property names to accurately reflect that they carry `Type.FullName` (namespace-qualified), not assembly-qualified names. Adds an appendix documenting why RemoteFactory manages its own serialization and how the type resolution pipeline works.

## What's New

- **Serialization internals appendix** (`docs/appendix/serialization.md`): Documents the three problems System.Text.Json can't solve (interface type preservation, DI-aware deserialization, shared object identity), the `Type.FullName`-based type resolution pipeline, and how the `ServiceAssemblies` type cache works.

## Breaking Changes

- **Wire format property rename**: `RemoteRequestDto.DelegateAssemblyType` renamed to `DelegateFullName`. `ObjectJson.AssemblyType` renamed to `TypeFullName`. These properties are serialized in the HTTP payload between client and server. **Both client and server must be updated together** (already required by ordinal serialization format).

## Bug Fixes

None

## Migration Guide

Rebuild and redeploy both client and server together. No code changes required in consuming applications — these are internal types. If you have custom middleware or logging that references `DelegateAssemblyType` or `AssemblyType` in JSON payloads, update those references to `DelegateFullName` and `TypeFullName`.

## Commits

- Rename `DelegateAssemblyType` to `DelegateFullName` and `AssemblyType` to `TypeFullName`
- Add `docs/appendix/serialization.md` explaining serialization internals
