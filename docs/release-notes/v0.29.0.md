# v0.29.0 - Authorization Target Parameter Support

**Release Date:** 2026-04-06
**NuGet:** [Neatoo.RemoteFactory 0.29.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.29.0)

## Overview

Authorization methods can now receive the target entity on write operations (Insert, Update, Delete), enabling state-based authorization decisions. Also adds broad test coverage for type-based parameter matching on class factories.

## What's New

### Target Parameter in Authorization Methods

Authorization methods decorated with `[AuthorizeFactory(AuthorizeFactoryOperation.Write)]` can now declare a parameter matching the entity type. On write operations, the generator passes the target entity to the authorization method so it can inspect entity state:

```csharp
public class EmployeeAuthorization
{
    [AuthorizeFactory(AuthorizeFactoryOperation.Write)]
    public bool CanWrite(IEmployee target)
    {
        // Deny writes to locked employees
        return target.Status != "Locked";
    }
}
```

When an authorization method has a target parameter:

- **Write operations** (Insert, Update, Delete) pass the target entity to the auth method
- **Read operations** (Create, Fetch) are unaffected -- the target auth does not apply
- **`CanInsert`/`CanUpdate`/`CanDelete`/`CanSave`** methods are **not generated** on the factory interface, since authorization requires the entity instance — *updated in [v1.3.0](v1.3.0.md): `CanSave(T target)` overload is now generated and runs target-aware authorization*
- **`CanCreate`/`CanFetch`** methods are still generated when Read authorization is parameterless

### Type-Based Parameter Matching on Class Factories

Authorization methods with typed parameters are now broadly tested for class factories. The generator matches auth method parameters **by type** to factory method parameters:

```csharp
public class OrderAuthorization
{
    [AuthorizeFactory(AuthorizeFactoryOperation.Fetch)]
    public bool CanFetch(Guid orderId)
    {
        // Check if user has access to this specific order
        return orderId != RestrictedOrderId;
    }
}
```

When `Fetch(Guid id)` is called, the generator passes `id` to `CanFetch(Guid orderId)` by matching the `Guid` type.
