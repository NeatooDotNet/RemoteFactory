# v10.11.1 - Nullable typeof() Fix

**Release Date:** 2026-01-18
**NuGet:** [Neatoo.RemoteFactory 10.11.1](https://nuget.org/packages/Neatoo.RemoteFactory/10.11.1)

## Overview

Fixes a compilation error (CS8639) when using nullable reference type properties with ordinal serialization.

## What's New

None.

## Breaking Changes

None.

## Bug Fixes

### Ordinal Serialization with Nullable Reference Types

Fixed CS8639 compilation error where the generator output `typeof(List<string>?)` for nullable reference type properties. The `?` suffix is invalid in `typeof()` expressions because nullability is a compile-time annotation only.

**Before:** Classes with nullable collection properties like `List<string>?` would fail to compile:

```csharp
// Generated code (invalid)
public static Type[] PropertyTypes { get; } = new[]
{
    typeof(System.Collections.Generic.List<string>?),  // CS8639 error
};
```

**After:** Nullable annotations are stripped from type names in `typeof()` expressions:

```csharp
// Generated code (valid)
public static Type[] PropertyTypes { get; } = new[]
{
    typeof(System.Collections.Generic.List<string>),  // Correct
};
```

## Commits

- fix: strip nullable annotation from typeof() in ordinal PropertyTypes array
