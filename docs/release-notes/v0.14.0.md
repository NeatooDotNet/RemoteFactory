# v0.14.0 - CanSave on IFactorySave\<T\>

**Release Date:** 2026-02-26
**NuGet:** [Neatoo.RemoteFactory 0.14.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.14.0)

## Overview

Adds `CanSave()` to the `IFactorySave<T>` interface so consumers can check save authorization without knowing the concrete factory type. The generator wires an explicit interface bridge that delegates to the concrete `CanSave` method when authorization is configured, or returns `Authorized(true)` by default.

## What's New

### IFactorySave\<T\>.CanSave()

The `IFactorySave<T>` interface now includes a `CanSave()` method, enabling authorization checks through the interface:

```csharp
var factorySave = serviceProvider.GetRequiredService<IFactorySave<Employee>>();
var authorized = await factorySave.CanSave();
if (authorized.HasAccess)
{
    await factorySave.Save(employee);
}
```

When a factory has authorization configured (e.g., `[CanSave]` methods), the generated explicit interface implementation delegates to the concrete method. When no authorization is configured, it returns `Authorized(true)` by default.

### Improved Documentation

The factory-operations draft documentation now includes "why" explanations for:
- Collection factories and persistence-aware collections
- Lifecycle hooks and their cross-cutting use cases
- CancellationToken threading through the pipeline
- Parameter ordering conventions (value, service, cancellation)
