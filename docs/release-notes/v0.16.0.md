# v0.16.0 - Hosted Blazor WebAssembly

**Release Date:** 2026-02-28
**NuGet:** [Neatoo.RemoteFactory 0.16.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.16.0)

## Overview

Converts all example and design projects to hosted Blazor WebAssembly, so each application starts with a single `dotnet run` on the server project. Eliminates hardcoded localhost URLs and CORS configuration.

## What's New

### Hosted Blazor WebAssembly

All four application groups (Design, Person, OrderEntry, Reference App) now use hosted WASM:

- **Single startup**: `dotnet run --project <Server>` starts both server and client
- **No hardcoded URLs**: Clients use `builder.HostEnvironment.BaseAddress` (same origin)
- **No CORS needed**: Server and client share the same origin in hosted mode
- **Documentation updated**: Getting started guide, skill content, and READMEs reflect the hosted pattern

### Project rename

`PersonApp` renamed to `Person.Client` for consistency with other client project names (`Design.Client.Blazor`, `OrderEntry.BlazorClient`).
