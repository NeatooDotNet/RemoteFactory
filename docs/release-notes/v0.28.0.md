# v0.28.0 - Event Scope Initializers & Pipeline Trace Logging

**Release Date:** 2026-04-03
**NuGet:** [Neatoo.RemoteFactory 0.28.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.28.0)

## Overview

Introduces `IEventScopeInitializer` for propagating ambient context (tenant context, user identity, etc.) from the request scope to event handler scopes. Also adds fine-grained `Trace`-level logging across the remote delegate pipeline.

## What's New

### Event Scope Initializers

- **`IEventScopeInitializer` interface** -- New extensible mechanism for propagating ambient context from the request scope to event handler scopes. Solves the multi-tenant problem where `[Remote, Event]` handlers lose tenant context (connection strings, tenant ID) because they run in isolated DI scopes.
- **`AddRemoteFactoryEventScopeInitializer` extension method** -- Register custom initializers to propagate any scoped state:
  ```csharp
  services.AddRemoteFactoryEventScopeInitializer((parentScope, childScope) =>
  {
      var parentTenant = parentScope.GetService<ITenantContext>();
      var childTenant = childScope.GetRequiredService<TenantContext>();
      if (parentTenant != null)
      {
          childTenant.TenantId = parentTenant.TenantId;
          childTenant.ConnectionString = parentTenant.ConnectionString;
      }
  });
  ```
- **Built-in correlation context propagation** -- `ICorrelationContext.CorrelationId` is now propagated via the built-in `CorrelationContextScopeInitializer` (replaces hardcoded generation). Registered automatically in Server/Logical modes.
- **Error resilience** -- Each initializer is wrapped in an individual try/catch with logging. A failing initializer does not prevent the event handler from running.

### Pipeline Trace Logging

- **Pipeline trace logging** -- 14 new `[LoggerMessage]` methods (event IDs 8001--8021) at `LogLevel.Trace` instrument every step of the remote request pipeline: request deserialization, delegate resolution, invocation, async awaiting, and response serialization.
- **Per-step timing** -- Each pipeline phase is independently timed via `Stopwatch`, so you can see exactly where time is spent (e.g., "Target deserialized in 12ms, delegate completed in 45ms, response serialized in 8ms").
- **Serializer trace logging** -- `NeatooJsonSerializer` now logs individual target and parameter deserialization with type names, JSON lengths, and elapsed time, plus serialize start/complete with declared vs. runtime type information.
- **Separate logger category** -- All trace logs use the `RemoteFactory.Trace` category, so they can be enabled independently of the existing `RemoteFactory.Server` logs via standard logging configuration.

### Enabling trace logging

Add to your `appsettings.Development.json`:

```json
{
  "Logging": {
    "LogLevel": {
      "RemoteFactory.Trace": "Trace"
    }
  }
}
```

## Breaking Changes

None. All new logging is additive and defaults to disabled (`Trace` level).

## Bug Fixes

### Parameter DTO Registration for IL Trimming

- **Fixed:** The source generator now registers plain DTO types used as **parameters** to factory methods (not just return types) in `DtoConstructorRegistry`. Previously, parameter DTOs like `AdminUpdateUserRequest` were not registered, causing IL trimming on Blazor WASM to strip their property metadata â€” resulting in empty JSON objects being serialized to the server. Affects `[Execute]` methods most commonly, but the fix covers all factory method types.
