# v0.19.0 - Trim-Safe Ordinal Serialization

**Release Date:** 2026-03-07
**NuGet:** [Neatoo.RemoteFactory 0.19.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.19.0)

## Overview

Fixes IL2026 build errors in generated ordinal converters when domain assemblies use `<IsTrimmable>true</IsTrimmable>` with `<TreatWarningsAsErrors>True</TreatWarningsAsErrors>`.

## What's New

None

## Breaking Changes

None

## Bug Fixes

- **Generated ordinal converters are now trim-safe**: The generator previously emitted `JsonSerializer.Serialize<T>()` and `JsonSerializer.Deserialize<T>()` calls in generated `*.Ordinal.g.cs` files. These generic overloads are marked `[RequiresUnreferencedCode]`, producing IL2026 warnings that become errors when `TreatWarningsAsErrors` is enabled. The generator now emits the trim-safe `GetTypeInfo`-based overloads (`JsonSerializer.Deserialize(ref reader, options.GetTypeInfo(typeof(T)))` and `JsonSerializer.Serialize(writer, value, options.GetTypeInfo(typeof(T)))`). Same runtime behavior, no IL2026 warnings.

- **Dead code removed**: Removed unused `GenerateOrdinalSerialization()` method from `FactoryGenerator.cs` that also contained trim-unsafe API calls.
