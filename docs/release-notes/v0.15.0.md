# v0.15.0 - Version Reset to v0.x Pre-release

**Release Date:** 2026-02-26
**NuGet:** [Neatoo.RemoteFactory 0.15.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.15.0)

## Overview

Resets the package version scheme from v10.x to v0.x to correctly reflect pre-release status. All prior v10.x NuGet packages have been unlisted.

## What's New

### Version scheme change

RemoteFactory was previously versioned as v10.x.0, which implied a mature, stable release. Since the package is still in active pre-release development with only the author using it, the version has been reset to v0.x.0.

The mapping is straightforward: v10.1.1 → v0.1.1, v10.2.0 → v0.2.0, ..., v10.14.0 → v0.14.0.

All prior NuGet packages (v9.x, v10.x, 2025.x-beta) have been unlisted from nuget.org. All git tags have been removed.

## Breaking Changes

- **Package version reset**: Update your NuGet references from `Neatoo.RemoteFactory` 10.14.0 to 0.15.0
- **Package ID unchanged**: The package names `Neatoo.RemoteFactory` and `Neatoo.RemoteFactory.AspNetCore` remain the same

## Bug Fixes

None

## Migration Guide

Update your package references:

```xml
<!-- Before -->
<PackageReference Include="Neatoo.RemoteFactory" Version="10.14.0" />
<PackageReference Include="Neatoo.RemoteFactory.AspNetCore" Version="10.14.0" />

<!-- After -->
<PackageReference Include="Neatoo.RemoteFactory" Version="0.15.0" />
<PackageReference Include="Neatoo.RemoteFactory.AspNetCore" Version="0.15.0" />
```

No code changes required. The API is identical to v10.14.0.
