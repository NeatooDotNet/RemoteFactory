# v0.21.1 - Fix [DynamicDependency] Trimming on Multi-Overload Factories

**Release Date:** 2026-03-08
**NuGet:** [Neatoo.RemoteFactory 0.21.1](https://nuget.org/packages/Neatoo.RemoteFactory/0.21.1)

## Overview

Fixes a bug where factory classes with multiple overloads were trimmed during IL trimming when the first overload had no callers in the published app.

## What's New

None

## Breaking Changes

None

## Bug Fixes

- **Fixed factories trimmed when first overload has no callers.** The generator emitted `[DynamicDependency]` only on the first interface method of each factory. If that method had no callers in the WASM app, the trimmer removed the method and its attribute, causing the entire factory class to disappear. Now `[DynamicDependency]` is emitted on every interface method so any surviving caller preserves the factory.

## Commits

- Emit `[DynamicDependency]` on every factory interface method, not just the first
