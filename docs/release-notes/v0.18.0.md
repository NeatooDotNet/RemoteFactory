# v0.18.0 - Auth Type Trimming Fix & IsTrimmable

**Release Date:** 2026-03-06
**NuGet:** [Neatoo.RemoteFactory 0.18.0](https://nuget.org/packages/Neatoo.RemoteFactory/0.18.0)

## Overview

Fixes `[AuthorizeFactory<T>]` types being stripped by the IL trimmer, and introduces `<IsTrimmable>true</IsTrimmable>` as the recommended way for domain assemblies to opt into trimming.

## What's New

### Auth Type Auto-Registration

The generator now emits explicit `services.TryAddTransient<IFooAuth, FooAuth>()` registrations in `FactoryServiceRegistrar` for every `[AuthorizeFactory<T>]` type. This creates static references that the IL trimmer preserves.

Previously, auth types (often `internal` classes) were only discovered at runtime via `RegisterMatchingName` reflection. When the domain assembly was trimmed, the trimmer saw no static references and removed them — causing "NoServiceRegistered" errors on the client when `Can*` methods tried to resolve the auth service.

The concrete type is resolved at compile time by searching the compilation for a class that implements the auth interface. This works regardless of naming conventions.

### IsTrimmable on Domain Assemblies

Domain model projects should now declare themselves as trimmable:

```xml
<PropertyGroup>
  <IsTrimmable>true</IsTrimmable>
</PropertyGroup>
```

This replaces the previous approach of adding `<TrimmableAssembly>` entries in each consuming client project. The library author declares trimmability once; consuming projects only need the `RuntimeHostConfigurationOption` for the `IsServerRuntime` feature switch.

## Breaking Changes

None

## Bug Fixes

- **Auth types survive IL trimming**: `[AuthorizeFactory<T>]` implementation classes are no longer stripped when the domain assembly is trimmed. The generator creates static references via explicit DI registration.

## Migration Guide

### Recommended: Add IsTrimmable to your domain project

In your domain model `.csproj`:

```xml
<PropertyGroup>
  <IsTrimmable>true</IsTrimmable>
</PropertyGroup>
```

Then remove the corresponding `<TrimmableAssembly Include="YourDomain" />` from your client `.csproj` — it's no longer needed.

### RegisterMatchingName and Trimming

`RegisterMatchingName` uses reflection, so types only discovered through it may be trimmed. Factory auth types are now handled automatically. For other convention-registered services, either register them explicitly or use `[DynamicDependency]` to preserve them from trimming.
