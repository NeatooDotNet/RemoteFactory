# v10.11.2 - Nullable Value Type Syntax Fix

**Release Date:** 2026-01-18
**NuGet:** [Neatoo.RemoteFactory 10.11.2](https://nuget.org/packages/Neatoo.RemoteFactory/10.11.2)

## Overview

Fixes compilation errors when using nullable value type properties (`int?`, `DateTime?`, `Guid?`, `decimal?`) with ordinal serialization.

## What's New

None.

## Breaking Changes

None.

## Bug Fixes

### Ordinal Serialization with Nullable Value Types

Fixed CS1525/CS1003/CS0747/CS0119 compilation errors where the generator produced invalid cast syntax for nullable value types in `FromOrdinalArray()`.

**Root Cause:** For nullable value types like `int?`, Roslyn's `ToDisplayString()` returns `"int?"` even after calling `WithNullableAnnotation(NotAnnotated)` because `int?` is actually `Nullable<int>` - a different underlying type. The renderer then added another `?`, producing invalid `(int??)` syntax.

**Before:** Records with nullable value type properties would fail to compile:

```csharp
// User code
[Factory]
[Create]
public partial record MyRecord(string Name, int? Value, DateTime? Date);

// Generated code (invalid)
public static object FromOrdinalArray(object?[] values)
{
    return new MyRecord(
        (string)values[0]!,
        (int??)values[1],    // CS1525: Invalid expression term
        (System.DateTime??)values[2]  // CS1525: Invalid expression term
    );
}
```

**After:** Type strings are normalized to prevent double nullable suffixes:

```csharp
// Generated code (valid)
public static object FromOrdinalArray(object?[] values)
{
    return new MyRecord(
        (string)values[0]!,
        (int?)values[1],
        (System.DateTime?)values[2]
    );
}
```

Also fixed the `Deserialize<T>` type parameter to use the nullable type for proper null handling:

```csharp
// Before: Deserialize<int> couldn't handle null values
var prop1 = JsonSerializer.Deserialize<int>(ref reader, options);

// After: Deserialize<int?> properly handles null
var prop1 = JsonSerializer.Deserialize<int?>(ref reader, options);
```
