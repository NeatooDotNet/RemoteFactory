# NF0403: Event method has no data parameters

## Description

Event methods marked with `[Event]` typically should have at least one non-service parameter to pass data to the handler. A method with only `[Service]` parameters and `CancellationToken` is unusual and may indicate a design issue.

| Property | Value |
|----------|-------|
| **Diagnostic ID** | NF0403 |
| **Severity** | Warning |
| **Category** | RemoteFactory.Usage |
| **Default** | Enabled |

## Example

The following code triggers NF0403:

```csharp
[Factory]
public partial class CleanupHandler
{
    // Warning NF0403: Event method with no non-service parameters
    [Event]
    public Task CleanupExpiredSessions(
        [Service] ISessionService sessionService,
        CancellationToken ct)
    {
        return sessionService.CleanupExpiredAsync(ct);
    }
}
```

## How to Fix

If the event is intended to operate on specific data, add a parameter:

```csharp
[Factory]
public partial class CleanupHandler
{
    // Correct: Has data parameter
    [Event]
    public Task CleanupExpiredSessions(
        DateTime olderThan,  // Data parameter
        [Service] ISessionService sessionService,
        CancellationToken ct)
    {
        return sessionService.CleanupOlderThanAsync(olderThan, ct);
    }
}
```

If this is intentional (e.g., a scheduled cleanup task), suppress the warning:

```csharp
[Factory]
public partial class CleanupHandler
{
    #pragma warning disable NF0403 // No data parameters is intentional
    [Event]
    public Task CleanupExpiredSessions(
        [Service] ISessionService sessionService,
        CancellationToken ct)
    {
        return sessionService.CleanupExpiredAsync(ct);
    }
    #pragma warning restore NF0403
}
```

## Why This Matters

Events are typically used to notify about something specific (an order, a user, an entity ID). The generated delegate's signature only includes data parameters:

```csharp
// Generated delegate has no parameters
public delegate Task CleanupExpiredSessionsEvent();
```

This signature makes it unclear what the event represents. Consider whether:

- The method should use `[Execute]` instead (if it's a command/query)
- A parameter should identify what triggered the event
- A scheduled job framework would be more appropriate
